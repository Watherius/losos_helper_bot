# Общие правила для всех агентов

> Каждый агент **обязан** прочитать этот файл перед началом работы.

## 1. Запрещено (для всех агентов)

- ❌ **`any`** — используй конкретный тип или `unknown` + type guard в TypeScript. В Python используй явные аннотации типов (например, `Any` из `typing` разрешено только в крайних случаях при интеграции с динамическими библиотеками).
- ❌ **`console.log`** в Node.js — используй логгер `pino` (через fastify логгер или экземпляр pino).
- ❌ **`print()`** в Python-коде продакшена — используй стандартное логирование FastAPI/uvicorn (`logging.getLogger("uvicorn")` или аналогичные логгеры).
- ❌ **Хардкод URL, портов, credentials** — всё через переменные окружения (`.env`), считываемые через `process.env` в Node.js или `Settings` (pydantic-settings) в Python.
- ❌ **Файл > 300 строк** — разбивай на модули.
- ❌ **Функция > 50 строк** — декомпозируй.
- ❌ **Установка пакетов глобально** (`npm install -g` или `pip install -g`).

## 2. Структура файла (TypeScript)

Порядок секций внутри каждого файла:
```
1. Импорты (node → npm → local)
2. Типы и интерфейсы
3. Константы
4. Основная логика / функции
5. Вспомогательные функции
6. Экспорт (если не inline)
```

## 3. Структура файла (Python)

Порядок секций внутри каждого файла:
```
1. Импорты (стандартные библиотеки -> сторонние библиотеки -> локальные модули)
2. Глобальные константы и настройки
3. Pydantic-модели / Схемы / DTO
4. Определение классов / SQLAlchemy моделей
5. Основные функции / роуты FastAPI
6. Вспомогательные функции (helpers)
```

## 4. Нейминг

| Что | Формат | Пример |
|-----|--------|--------|
| Файл утилиты / модуля | camelCase / snake_case | `dates.ts` / `redis_client.py` |
| Класс / Модель / Компонент | PascalCase | `Message` / `ResponseLog` |
| Переменная / Функция | camelCase (TS) / snake_case (Python) | `waMessageId` (TS) / `wa_message_id` (Python) |
| Тип / Интерфейс (TS) | PascalCase | `WebhookPayload` |
| Константа | UPPER_SNAKE | `BUFFER_TIMEOUT_SECONDS` |

## 5. Обработка ошибок

- **Python (FastAPI)**: выбрасывай `HTTPException(status_code=..., detail=...)` для ответов клиенту.
- **Node.js**: возвращай структурированные ошибки в соответствии с Fastify-стандартом или генерируй ошибки с соответствующим статус-кодом.
