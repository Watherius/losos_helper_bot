# Правила работы для AI Агентов (Workspace Rules)

## 1. Изменения базы данных и моделей (SQLAlchemy)
- ⚠️ **ОБЯЗАТЕЛЬНО**: Любые изменения в файле `backend-python/app/models.py` должны быть тщательно проверены.
- Так как проект использует автоматическое создание таблиц при запуске бэкенда через `Base.metadata.create_all` (в `app/db.py`), существующие колонки/таблицы не обновляются автоматически.
- При изменении схемы базы данных локально необходимо:
  1. Либо сбросить базу данных (если это возможно для локальной разработки) и перезапустить контейнеры.
  2. Либо применить ручные миграции SQL прямо в базе данных для добавления новых колонок/таблиц.
- Категорически запрещено ломать совместимость со старыми данными без предварительного согласования.

## 2. Проверка типов и линтинг перед сдачей задач
- ⚠️ **ОБЯЗАТЕЛЬНО**: После любого изменения кода (создания или редактирования файлов) AI-агент **обязан самостоятельно** запустить проверку типов в корне проекта:
  ```bash
  .\typecheck.bat
  ```
- Убедитесь, что ни TypeScript-компилятор, ни `mypy` не выдают ошибок. Любые ошибки типов должны быть исправлены агентом до завершения задачи и демонстрации пользователю.
